# Job-fair demo runbook

## Before doors open

1. Pull the exact DVC snapshot and run preflight.
2. Start Compose and wait for all four services to report healthy.
3. Warm the selected live model with a non-demo request.
4. Run all three presets once; do not save or replay their answers.
5. Use “Reset showcase” to clear browser-local decisions.

## Recruiter flow

Lead with the Dubai Marina preset and select **Find matching homes** once. Point out that successful interpretation immediately authorizes the validated brief, while **Edit brief** keeps correction available without a blocking confirmation screen. Let the cinematic trace transform into the completion takeover; explain the three restrained evidence metrics, then expand **View agent trace** to show the eight-node flow. Open the best match, compare the top two, show an unknown where present, enter one shared affordability scenario, copy the decision summary, and open print preview. Finish at `#/case-study` and the architecture diagram.

## Failure recovery

If the provider fails, show the calm error as intentional live-only behavior, confirm preflight, and retry the validated brief. Demonstrate cancellation only if useful; **Run again** never replays an earlier answer. If the map tiles fail, show the coordinate evidence fallback. Never switch to saved AI output.
